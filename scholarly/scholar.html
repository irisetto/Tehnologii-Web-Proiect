Zoo Web Manager - Scholarly HTML

Introducere

Scopul acestui raport este de a prezenta interfata responsive dezvoltata pentru o aplicație web menita de a gestiona informațiile despre animalele găzduite într-o grădină zoologică. Scopul acestei aplicații este de a facilita redarea și managementul datelor despre animalele găzduite, precum și de a permite utilizatorilor să efectueze căutări și filtrări complexe pentru a găsi informațiile dorite.

Cerința

Să se implementeze o aplicație Web menită a gestiona informațiile vizând o grădina zoologică. Se vor facilita redarea și managementul datelor despre animalele găzduite: denumire populară/științifică, descriere (de dorit, multilingvă), origine, statut (e.g., specie protejată, pe cale de dispariție, dăunătoare etc.), caracteristici (de exemplu, dependență de climă, periculozitate, proprietăți speciale), specii înrudite, dușmani naturali, imagini de interes, regim alimentar (vegetarian, carnivor, omnivor), mod de înmulțire și altele. Se vor pune la dispoziție mecanisme de căutare/filtrare/prezentare mulți-criteriala -- de pildă, afișarea tuturor mamiferelor cu blană de origine europeană ce pot fi dresate sau generarea unei liste de păsări/târâtoare capabile să supraviețuiască în condiții deșertice. Datele vor putea fi importate/exportate via fișiere disponibile în formatele JSON și XML. Funcționalitățile esențiale se vor oferi și de către un API REST/GraphQL.

Design

Aplicatia permite crearea unui cont de utilizator pentru angajatii unei gradini zoologice si gestionarea usoara a animalelor prezente in zoo. Unui utilizator ii este permis sa vizualizeze informatiii despre animalele prezente in gradina zoologica si sa le filtreze dupa orice categorie este nevoie. De asemenea un utilizator poate sa exporte sau sa importe un fisier json sau xml care descrie un animal.

Tehnologii utilizate

Paginile au fost create fără utilizarea unui framework. HTML este folosit pentru a structura conținutul și a crea elemente semantice. CSS este utilizat pentru a stiliza paginile și a le face mai atractive prin adăugarea de stiluri și efecte de animație. JavaScript este folosit pentru a adăuga funcționalitate in plus paginilor.

Arhitectura aplicației

Arhitectura acestei aplicații web este bazată pe o structură simplă de pagini, cu o navigare ușor de utilizat:

    • Pagina login - această pagină permite utilizatorilor să se autentifice și să acceseze funcționalitățile aplicației.
      Pagina register - această pagină permite utilizatorilor să isi creeze un cont.
      Pagina home - această pagină servește ca punct de intrare pentru utilizatorii care accesează aplicația dupa ce s-au logat
      Pagina animals - această pagină conține informații despre animalele din grădina zoologică, inclusiv denumirea populară și științifică, descriere, origine, statut, caracteristici, specii înrudite, dușmani naturali și imagini. Utilizatorii pot căuta și filtra animalele în funcție de diferite criterii.
      Pagina help - această pagină oferă utilizatorilor informații despre cum să utilizeze aplicația și cum să acceseze diferitele funcționalități.
      Pagina profile - această pagină permite utilizatorilor să-și vizualizezesi si gestioneze profilul (să-și modifice informațiile personale si/sau parola). De asemenea este posibil de a trimite un ticket catre un manager al unei sectiuni pentru a aduce in vederea tuturor o posibila problema vizand gradina zoologică.
      Pagina settings - această pagină permite utilizatorilor să modifice diferite setari din cadrul aplicatiei precum tema de fundal sau limba in care este prezentat textul. De asemenea, va fi posibil ca un utilizator sa importe sau exporte un fisier json sau xml care reprezinta un animal din gradina zoologica
      Pagina animal_id - această pagină permite utilizatorilor să vizualizeze informatii despre un anumit animal selectat.
      Pagina about - această pagină conține informații despre aplicatie si despre autorii proiectului

    • Fiecare pagină, cu excepția paginilor de login, register si landing, are un meniu navigațional care permite utilizatorilor să acceseze toate celelalte pagini. De asemenea, toate paginile au un footer decorativ care include logo-ul aplicatiei.
